## Brownfield Onboarding

The warm-start engagement flow for existing systems: a user brings an existing codebase or running system; the multi-agent system reverse-engineers a validated architecture model, assesses gaps and risks, and transitions the system into ongoing governance — all without requiring the user to author architecture artifacts themselves.

## Properties

| Attribute | Value |
|---|---|
| Trigger | User provides an existing target repository at entry point EP-G |
| Outcome | User receives a confirmed, versioned architecture model covering all ArchiMate layers, a risk-annotated gap assessment, and the system operating under continuous Phase G governance |

## Stages

| Stage | Consuming Stakeholder | Value Delivered | Key Business Services |
|---|---|---|---|
| 1 — System Discovery | User (engagement owner) | Preliminary scan report: inferred entity inventory, technology fingerprint, and identified scan gaps requiring user clarification | BSV-002 (App & Technology Architecture), BSV-005 (Requirements Management) |
| 2 — Architecture Reconstruction | User (engagement owner) | Reconstructed multi-layer architecture model (business + application + technology) with all inferred entities annotated with evidence sources | BSV-001 (Business Architecture), BSV-002 |
| 3 — Gap & Risk Assessment | User (engagement owner) | Risk-annotated gap analysis: missing capabilities, underdeveloped services, safety concerns, and prioritised remediation recommendations | BSV-001, BSV-004 (Safety Governance) |
| 4 — Model Confirmation | User (engagement owner) | User-validated architecture baseline: inferred assumptions confirmed or corrected, CQs resolved, model promoted to draft-approved status | BSV-009 (User Decisions), BSV-005 |
| 5 — Governance Activation | User (engagement owner) / Architecture Board | System operating under continuous Phase G governance: change-impact assessment active, algedonic channel live, enterprise patterns available for promotion | BSV-003 (Project Coordination), BSV-004, BSV-001 |
