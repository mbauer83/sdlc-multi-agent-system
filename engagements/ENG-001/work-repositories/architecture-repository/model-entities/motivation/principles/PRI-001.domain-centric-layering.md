## Domain-Centric Layering

## Properties

| Attribute | Value |
|---|---|
| Statement | Dependencies flow strictly inward: Infrastructure → Application → Domain → Common; never reversed. |
| Rationale | Keeps domain logic testable without infrastructure and prevents framework coupling from leaking into business rules. Enables the LLM and orchestration layers to be swapped without touching domain logic (see CST-001). |
| Implications | `src/common/` may be imported by all layers. `src/events/` and `src/agents/` may import from `src/models/` but never from `src/orchestration/` or `src/dashboard/`. No framework import (`langgraph`, `pydantic_ai`, `fastapi`) in `src/models/` or `src/common/`. |
