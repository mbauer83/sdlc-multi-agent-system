## EventStore Primacy

## Properties

| Attribute | Value |
|---|---|
| Statement | `workflow.db` is the sole canonical state; no agent holds private mutable state between invocations. |
| Rationale | Makes the full engagement history auditable and replayable. Enables resumable state (REQ-004) and end-to-end traceability (GOL-002). Eliminates hidden state that would survive restarts or cause divergence between agent instances. |
| Implications | Every mutation to workflow state, every artifact write, and every user interaction must produce an EventStore event. Agents receive `WorkflowState` reconstructed from events at each invocation; they do not retain state across calls. |
