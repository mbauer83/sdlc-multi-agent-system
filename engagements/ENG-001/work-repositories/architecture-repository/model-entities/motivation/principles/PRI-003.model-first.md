## Model-First

## Properties

| Attribute | Value |
|---|---|
| Statement | Entity files exist before any diagram element that references them; diagram aliases must resolve to ModelRegistry entries. |
| Rationale | Ensures diagrams are always grounded in the architecture model. Prevents diagrams from diverging silently from the entity corpus. Enables `validate_diagram` to check all aliases before rendering. |
| Implications | A diagram element with no backing entity is a hard error (ALG-C03). Reverse-architecture skills write entity files first; diagrams are produced afterward. Stage 5 code changes that introduce new components must create or update the corresponding entity file before the code change lands. |
