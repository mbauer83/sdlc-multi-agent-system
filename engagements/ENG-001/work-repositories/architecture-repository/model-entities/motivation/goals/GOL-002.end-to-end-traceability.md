## End-to-End Traceability

Every artifact produced by the system is traceable to: (a) a requirement that motivated it, (b) the agent and skill that produced it, (c) the sprint in which it was produced, and (d) any user-provided input (CQ answer or uploaded file) that informed it. The EventStore is the mechanism for this traceability.

## Properties

| Attribute | Value |
|---|---|
| Time Horizon | Long |
| Measurable | Yes |

## Notes

Measurability: every `artifact.created` event in the EventStore must carry `produced_by_skill` and be reachable from a requirement via connection files.
