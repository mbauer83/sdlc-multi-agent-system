## Architecture Documentation Debt

Brownfield systems commonly lack architecture models. Key design decisions are undocumented, onboarding new team members is slow and error-prone, and there is no baseline from which to assess change impact. Reverse-engineering the architecture model from code and infrastructure is manual and time-consuming.
