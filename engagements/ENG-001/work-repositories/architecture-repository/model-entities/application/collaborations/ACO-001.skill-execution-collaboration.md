## Skill Execution Collaboration

**Structural element only.** Groups the application services that act together within the Skill Execution interaction (AIA-001). An ApplicationCollaboration in ArchiMate is the *active structure* counterpart of an ApplicationInteraction — it aggregates the participants; the interaction is the behavioral specification. This element does NOT itself realize BPR-002; that realization belongs to AIA-001.

Participating services:
| Service | Contribution within AIA-001 |
|---|---|
| ASV-001 Agent Invocation Service | Agent invoked with 4-layer prompt; skill steps executed; events emitted |
| ASV-002 Artifact I/O Service | Input artifacts read via `read_artifact()`; outputs written via `write_artifact()` |
| ASV-005 Learning Service | Learnings retrieved at Step 0.L; new entries generated at feedback-loop close |

Each participating service acts through its corresponding port (AIF-002 LLMClientPort, AIF-004 ArtifactReadWriterPort, AIF-006 LearningStorePort).
