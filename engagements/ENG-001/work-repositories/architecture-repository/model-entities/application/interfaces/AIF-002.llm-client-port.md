## LLMClientPort

Conceptual LLM provider boundary in the application layer. The concrete provider is selected via PydanticAI's `provider:model-id` string convention (e.g. `anthropic:claude-sonnet-4-6`, `openai:gpt-4o`, `ollama:llama3.1`). Provider selection and model identity are configured through `LLMConfig` (DOB-015), which is loaded at composition root and passed into `build_agent()`.

PydanticAI `Agent` is the implementation: it wraps the provider SDK (Anthropic, OpenAI, Ollama, etc.) and handles structured output, tool use, and result validation. Provider SDKs are optional extras in `pyproject.toml`; only the configured provider's extra needs to be installed.

For API-free testing, PydanticAI's built-in `TestModel` provides deterministic structured output without any LLM calls — no separate stub protocol required.

## Properties

| Attribute | Value |
|---|---|
| Implemented by | PydanticAI `Agent` (provider selected via `LLMConfig`) |
| Provider config | `LLMConfig` (DOB-015) |
| Test double | PydanticAI `TestModel` (built-in; no LLM API calls) |
| Key constraint | CST-001 (No Framework Lock-In) |
