## ArtifactReadWriterPort

Port (typing.Protocol) for ERP entity file I/O. Path-constrained per agent role. Validates ERP frontmatter and §content/§display structure on every write. Updates ModelRegistry (APP-002) synchronously. Implements tool-transparent event emission — no skill instruction required.

Method signatures:
- `read(id_or_path: str, mode: str) → str`
- `write(path: str, content: str, *, upload_refs: list[str] | None = None) → ArtifactRecord`
- `list(**filter) → list[ArtifactRecord]`
- `search(query: str, **filter) → list[tuple[ArtifactRecord, str]]`

**Tool-transparent event emission (all from inside write()):**

Common case — `write(path, content)` (no upload_refs):
1. `artifact.created` or `artifact.updated` — auto-detects new vs existing file (always emitted)
2. `source_evidence` auto-extracted from `[inferred: <source>]` annotations in the written content — no agent parameter needed; relies on the annotation contract already required by reverse-arch skill steps

Full case — `write(path, content, upload_refs=[...])`:
3. `entity.confirmed` — emitted when `active_skill_id` ∈ `{SA-REV-*, SWA-REV-*}`; `confirmation_method` = `"user"` if a `cq.answered` event exists for this skill invocation, `"auto"` otherwise
4. `file.referenced` — emitted for each entry in `upload_refs`; agent passes this list explicitly only when a CQ answer cited a user-uploaded file

`upload_refs` is the only agent-visible parameter; all other emission is fully automatic.

## Properties

| Attribute | Value |
|---|---|
| Protocol location | `src/agents/protocols.py` |
| Path constraint | Enforced per agent role at construction time; instances constructed in AgentFactory (APP-005) tool registration |
| ERP validation | Frontmatter + §content + §display structure on every write |
| ModelRegistry sync | Synchronous upsert on every write call |
