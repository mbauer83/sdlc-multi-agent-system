## EngagementSession

**Architectural responsibility: session lifecycle and EventStore bootstrap.** Handles all pre-graph work (config loading, EventStore init, state replay) and the outer user-interaction loop. Does not define nodes or routing — that is LangGraph Orchestrator (APP-016). Think: APP-017 is the loader and lifecycle manager; APP-016 is the engine it starts.

`src/orchestration/session.py`. Top-level entry point for running an engagement. Manages the outer user interaction loop.

Startup sequence:
1. Load `engagements-config.yaml` → parse into `Engagement` (DOB-002)
2. Bootstrap EventStore (create schema if new; open connection)
3. Call `EventStore.replay_from_latest_snapshot()` (falls back to full `replay()`) to reconstruct `WorkflowState` (DOB-009); populate initial `SDLCGraphState` (DOB-012) from the reconstructed state
4. Emit `engagement.started` + `create_snapshot("engagement.started")`
5. Invoke `graph.stream()` (APP-016)
6. Detect sprint boundaries: emit `sprint.started` when PM returns `next_action == "invoke_specialist"` for the first skill in a new sprint

Outer loop: polls for user input, routes answers from UserInteractionOrchestrator (APP-018), and handles clean exit.
