## EventStore

`src/events/event_store.py`. ACID-guaranteed append-only event log backed by SQLite. Manages two tables: `events` (append-only; never mutated) and `snapshots` (point-in-time `WorkflowState` serialisations). The sole canonical write path for workflow state — no agent accesses `workflow.db` directly via sqlite3.

Public API:
- `append(payload, actor, cycle_id, correlation_id) → EventEnvelope` — Pydantic-validates and appends; increments sequence; best-effort YAML export on each call
- `replay_state() → WorkflowState` — full replay from event 0; writes snapshot on completion; used for integrity check and disaster recovery
- `replay_from_latest_snapshot() → tuple[WorkflowState, int]` — O(delta) fast path: loads latest snapshot, applies only delta events via `apply_events_to_state()`; returns `(state, delta_count)`
- `current_state() → WorkflowState` — reads from snapshot table (fast path); falls back to `replay_state()` if absent
- `create_snapshot(trigger: str) → str` — force-writes a snapshot at the current event head; returns `snapshot_at` tag; called explicitly by orchestration nodes at lifecycle boundaries
- `check_snapshot_interval() → bool` — returns True when ≥100 events have been appended since the last snapshot; called after every specialist node to decide periodic snapshot
- `query(event_type, cycle_id, correlation_id, limit) → list[dict]` — read path for agent inspection; returns matching event rows as dicts
- `export_yaml() → int` — exports all SQLite events lacking a YAML file to `workflow-events/`; called at sprint close
- `check_integrity() → list[str]` — validates JSON payloads and sequence continuity; returns anomaly descriptions

Snapshot triggers: situative at `engagement.started`, `gate.evaluated/passed`, `sprint.close`, `engagement.completed`; periodic every 100 events (checked via `check_snapshot_interval()`).
