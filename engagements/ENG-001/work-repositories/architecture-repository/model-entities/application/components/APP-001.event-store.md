## EventStore

`src/events/event_store.py`. ACID-guaranteed append-only event log backed by SQLite. Manages two tables: `events` (append-only; never mutated) and `snapshots` (point-in-time `WorkflowState` serialisations). The sole canonical write path for workflow state — no agent accesses `workflow.db` directly via sqlite3.

Public API:
- `record_event(event: WorkflowEvent) → None` — validates and appends; increments sequence
- `replay() → WorkflowState` — full replay from event 0; used for integrity check and disaster recovery
- `replay_from_latest_snapshot() → tuple[WorkflowState, int]` — fast path: deserialises latest snapshot + replays delta events; returns `(state, resume_sequence)`
- `create_snapshot(trigger: str) → str` — serialises current `WorkflowState` to JSON; inserts into `snapshots`; returns `snapshot_id`
- `check_snapshot_interval() → bool` — true if events since last snapshot ≥ snapshot interval (default 100)
- `check_integrity()` — validates JSON payloads, sequence gaps, YAML vs SQLite consistency
- `query_events(**filter) → list[WorkflowEvent]` — read path for replay and audit

Snapshot triggers: mandatory at `engagement.started`, `gate.evaluated/passed`, `sprint.close`; periodic every 100 events.
