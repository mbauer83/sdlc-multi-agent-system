## LangGraph Orchestrator

**Architectural responsibility: the state machine.** Defines all nodes, edges, and conditional routing logic. Does not handle bootstrapping, configuration loading, or the outer user-interaction loop — that is EngagementSession (APP-017). Loaded by EngagementSession at startup; runs as the compiled graph for the duration of the engagement.

`src/orchestration/graph.py`. Builds and compiles the stateful LangGraph graph that implements the ADM sprint lifecycle. Entry point: `build_sdlc_graph() → CompiledGraph`.

**Nodes:**
- `pm_node` — PM Agent decision cycle
- `sa_node`, `swa_node`, `do_node`, `de_node`, `qa_node`, `po_node`, `sm_node`, `csco_node` — specialist agent nodes
- `gate_check_node` — evaluates gate outcomes; emits `phase.transitioned` on pass
- `cq_user_node` — suspends graph; emits `phase.suspended`/`phase.resumed`
- `algedonic_handler_node` — routes algedonic signals; emits `algedonic.escalated`
- `sprint_close_node` — emits `sprint.close` + `create_snapshot("sprint.close")`
- `review_processing_node` — processes review items; emits `review.sprint-closed`
- `engagement_complete_node` — emits `engagement.completed` + `create_snapshot("engagement.completed")`

Each node receives `event_store: EventStorePort` directly (not only via AgentDeps) to emit lifecycle events without delegating to an agent. `SDLCGraphState` (DOB-012) is the shared state object.
