## UserInteractionOrchestrator

**Role: read-side event monitor and graph signal router.** Distinct from UserInputGateway (APP-021), which is the write-side HTTP endpoint that accepts user submissions and writes them to EventStore. This component only reads EventStore and signals the graph — it never accepts HTTP requests or writes events.

`src/orchestration/user_interaction.py`. Runs as an async monitor alongside the graph. Bridges completed user actions (already recorded by UserInputGateway) to the running LangGraph graph nodes that are waiting for them.

Responsibilities:
- Watch EventStore for `cq.answered` events (emitted by UserInputGateway after user submits an answer via Dashboard)
- Watch EventStore for `review.submitted` events
- Route CQ answers to the raising agent's suspended node; emit `cq.routed` event
- Signal `review_processing_node` on review submission

This component decouples the graph execution loop from blocking user I/O. The graph suspends at `cq_user_node`; this orchestrator reactivates it when an answer arrives in EventStore.
