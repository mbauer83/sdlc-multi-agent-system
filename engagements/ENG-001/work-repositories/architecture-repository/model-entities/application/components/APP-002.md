## ModelRegistry

`src/common/model_registry.py`. Two-tier artifact index providing the ERP v2.0 entity lookup layer.

**Tier 1 — in-memory frontmatter cache:** `dict[str, ArtifactRecord]` for structured metadata filtering. Methods: `list_artifacts(**filter)`, `list_connections(**filter)`, `get_by_id(id)`, `upsert(record)`.

**Tier 2 — SQLite FTS5 full-text index** over artifact `§content` blocks. Method: `search(query: str, **filter) → list[tuple[ArtifactRecord, str]]` — returns ranked records + matched snippet; filters applied as post-FTS metadata constraints.

**Optional semantic tier:** enabled when corpus ≥ 50 artifacts and `sqlite-vec` is available. Embeddings stored in `model_registry.db` alongside FTS5 table; cosine similarity replaces BM25 ranking.

Both tiers updated synchronously on every `write_artifact` call. Thread-safe (RLock). FTS5 index persisted to `engagements/<id>/model_registry.db` (git-tracked). Rebuilt from files on cold start or integrity mismatch.
