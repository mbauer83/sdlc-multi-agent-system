## UserInputGateway

`src/dashboard/interaction.py`. The write-side entry point of the Dashboard — validates all user-submitted inputs and records them through EventStore. Acts as the system boundary between user HTTP actions and engagement state mutations.

Routes handled:
- `POST /queries/<cq_id>/answer` — validates answer text; calls `EventStore.record_event(cq.answered)`; optionally links upload_id
- `POST /uploads` — validates MIME type against allow-list; stores file at `engagements/<id>/user-uploads/<upload_id>/`; calls `EventStore.record_event(upload.registered)`
- `POST /review/submit` — validates all items decided; calls `EventStore.record_event(review.submitted)` with ReviewItem list payload

Security constraints:
- MIME allow-list enforced (no executable or archive types)
- Path-restricted (never touches work-repositories directly — all writes go through EventStore)
- CQ ID validated against open CQ list before accepting answer

Named "gateway" (ports-and-adapters term) because it is the sole authorised entry point for user-side mutations. No agent or orchestration component may bypass it to write user inputs directly.
