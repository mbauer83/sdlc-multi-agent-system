## TargetRepoManager

`src/sources/target_repo.py`. Manages local clones of target project repositories configured in `engagements-config.yaml`.

Public API:
- `clone_or_update(repo_id: str) → Path` — clones repo if absent; fetches + pulls if present
- `get_clone_path(repo_id: str) → Path` — returns `engagements/<id>/target-repos/<repo-id>/`
- `check_access(repo_id: str, agent_role: str) → bool` — enforces read-only vs read-write per agent role
- `create_worktree(repo_id: str, branch: str) → Path` — creates git worktree for DE sprint isolation
- `get_primary_id() → str` — returns the `primary: true` repo ID

Backward-compatible: singular `target-repository` config treated as `target-repositories: [{id: "default", primary: true}]`.

Multi-repo: all `target_repo_tools` functions require explicit `repo_id`; default to primary repo when exactly one repo is registered.

## Properties

| Attribute | Value |
|---|---|
| Module | `src/sources/target_repo.py` |
| Config source | `engagements-config.yaml` (target-repositories list) |
| Local clone path | `engagements/<id>/target-repos/<repo-id>/` (.gitignored) |
| Write agents | DE Agent (code), DO Agent (IaC) |
| Read-only agents | SA, SwA, QA |
