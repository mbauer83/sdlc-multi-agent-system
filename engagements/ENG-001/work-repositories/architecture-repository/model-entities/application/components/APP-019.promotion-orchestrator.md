## PromotionOrchestrator

`src/orchestration/promotion.py`. Implements the enterprise promotion procedure defined in `framework/repository-conventions.md §12`.

Primary operation: `promote_entity(entity_id: str, engagement_id: str) → str`
1. Read entity file from engagement architecture repository
2. Assign globally unique enterprise ID from `enterprise-repository/governance-log/id-counters.yaml`
3. Perform deterministic reference sweep: replace old engagement-local ID with new enterprise ID across all connection files, diagram PUML aliases, and inline references in the engagement repository
4. Move entity file from engagement path to enterprise repository path
5. Emit `artifact.promoted` EventStore event; return new enterprise ID

Only Architecture Board members (STK-002) trigger this operation. Agents cannot call `promote_entity` directly — it is invoked by PM at engagement close during Architecture Board review.
