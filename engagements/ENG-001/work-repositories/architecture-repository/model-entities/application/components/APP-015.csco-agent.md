## CSCO Agent

`src/agents/csco.py`. PydanticAI agent for the Chief Safety & Compliance Officer role. All phases (gate reviews); produces Safety Constraint Overlay, STAMP/STPA analyses, incident records.

Tool set: `universal_tools` + `write_tools` (safety-repository bound) + gate vote authority on all phases.

Loadable skills: stamp-stpa-methodology, gate-phase-a, gate-phase-b, gate-phase-c, gate-phase-d, gate-phase-g, gate-phase-h, incident-response.

No gate skills for phases E or F — this is intentional. Phases E (Opportunities & Solutions) and F (Migration Planning) are planning/analysis phases without hard safety checkpoints; CSCO consults on them but does not hold a formal gate vote. Gate reviews concentrate at the architecture transition points (A→B, B→C, C→D) and governance phases (G, H).

Marked `safety-relevant: true` because this component is the primary safety gate authority — its gate vote (`veto`) can block any phase transition. Unavailability triggers ALG-014.
