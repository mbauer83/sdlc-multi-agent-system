## AlgedonicSignal

`src/models/algedonic.py`. Pydantic model for an algedonic escalation signal. Persisted in EventStore and tracked in `WorkflowState.last_algedonic`. Marked `safety-relevant: true` — algedonic signals can represent safety-critical conditions (S1 severity).

Fields:
| Field | Type | Description |
|---|---|---|
| `alg_id` | `str` | `ALG-NNN` assigned at raise time |
| `trigger_id` | `str` | Algedonic trigger code (e.g. `ALG-014`) per `algedonic-protocol.md` |
| `category` | `str` | Trigger category (Safety / Technical / Process / etc.) |
| `severity` | `Literal["S1","S2","S3"]` | S1 = immediate halt; S3 = monitor |
| `escalation_target` | `str` | Target of escalation (User, PM, CSCO, Architecture Board) |
| `raised_by` | `str` | Agent role that raised the signal |
| `raised_at` | `datetime` | UTC timestamp |
| `status` | `Literal["active","resolved","superseded"]` | Lifecycle state |
| `resolution_notes` | `str \| None` | Notes on how the signal was resolved |

## Properties

|---|---|
| Module | `src/models/algedonic.py` |
| Tracked in | `WorkflowState.last_algedonic` |
