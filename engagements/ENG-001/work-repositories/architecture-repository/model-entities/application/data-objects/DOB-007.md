## GateOutcome

`src/models/gate.py`. Pydantic model for a phase gate evaluation result. Tracked in `WorkflowState.gate_outcomes` keyed by `gate_id`. Contains sub-model `GateVote` (one per voting agent).

Fields:
| Field | Type | Description |
|---|---|---|
| `gate_id` | `str` | `GATE-<PHASE>-<N>` assigned by PM |
| `phase_from` | `str` | Phase exiting (e.g. `"B"`) |
| `phase_to` | `str` | Phase entering (e.g. `"C"`) |
| `votes` | `list[GateVote]` | One vote per participating agent |
| `result` | `Literal["passed","failed","deferred"]` | Aggregated gate outcome |
| `evaluated_at` | `datetime` | UTC timestamp |
| `rationale` | `str` | PM rationale for final result |

**GateVote sub-model fields:**
| Field | Type | Description |
|---|---|---|
| `agent` | `str` | Voting agent role |
| `result` | `Literal["approved","veto","abstain"]` | Individual vote; veto = gate fails |
| `rationale` | `str \| None` | Vote rationale |

## Properties

| Attribute | Value |
|---|---|
| Module | `src/models/gate.py` |
| Tracked in | `WorkflowState.gate_outcomes` (dict keyed by gate_id) |
| Veto rule | Any single veto → gate result = "failed" |
