## WorkflowState

**Primary use: restore full engagement state from event history on startup or for disaster recovery.**

`src/events/replay.py`. The canonical in-memory representation of engagement state, rebuilt by replaying EventStore events. Also the snapshot payload: serialised to JSON in the `snapshots` table to speed up future startups. Distinct from `SDLCGraphState` (DOB-012) which carries only the minimal routing context LangGraph needs during one graph execution cycle. Serialised to JSON for `snapshots` table storage. Must be fully JSON-serialisable (no `Path` objects; use `str`).

Fields:
| Field | Type | Description |
|---|---|---|
| `current_phase` | `str` | Current ADM phase identifier |
| `current_sprint` | `str \| None` | Current sprint ID |
| `phase_visit_counts` | `dict[str, int]` | Number of times each phase has been entered (ADM non-linearity) |
| `open_cqs` | `list[ClarificationRequest]` | All open CQs (status="open") |
| `pending_handoffs` | `list[HandoffRecord]` | Unacknowledged handoffs |
| `baselined_artifacts` | `dict[str, str]` | artifact_id → version at last baseline |
| `gate_outcomes` | `dict[str, GateOutcome]` | gate_id → outcome (all evaluated gates) |
| `last_algedonic` | `AlgedonicSignal \| None` | Most recent active algedonic signal |
| `registered_uploads` | `list[str]` | Upload IDs received in this engagement |
| `upload_reference_map` | `dict[str, list[str]]` | upload_id → artifact_ids that reference it |
| `snapshot_sequence` | `int \| None` | Sequence of triggering event; None if built by full replay |

## Properties

| Attribute | Value |
|---|---|
| Module | `src/events/replay.py` |
| Built by | `EventStore.replay()` (full) or `EventStore.replay_from_latest_snapshot()` (fast) |
| Serialised to | `snapshots.state_json` (JSON) |
| Constraint | No Path objects — all paths stored as str |
