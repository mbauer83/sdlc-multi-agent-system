## ClarificationRequest

`src/models/cq.py`. Pydantic model representing a single Clarification Request raised by a specialist agent. Persisted via EventStore events and tracked in `WorkflowState.open_cqs`.

Fields:
| Field | Type | Description |
|---|---|---|
| `cq_id` | `str` | `<ROLE>-CQ-NNN` assigned at raise time |
| `raising_agent` | `str` | Agent role that raised the CQ |
| `skill_id` | `str` | Active skill at time of raise |
| `phase` | `str` | Current ADM phase |
| `question_text` | `str` | Specific, actionable question for the user |
| `blocking` | `bool` | True = sprint blocker; PM tracks as blocker |
| `blocks_task` | `str \| None` | Task ID blocked (if blocking: true) |
| `status` | `Literal["open","answered","withdrawn"]` | Lifecycle state |
| `answer_text` | `str \| None` | User-provided answer |
| `upload_refs` | `list[str]` | IDs of user-uploaded files attached to answer |
| `raised_at` | `datetime` | UTC timestamp |
| `answered_at` | `datetime \| None` | UTC timestamp of answer |

## Properties

| Attribute | Value |
|---|---|
| Module | `src/models/cq.py` |
| Tracked in | `WorkflowState.open_cqs` |
| Events produced | `cq.raised`, `cq.answered` (via UserInputGateway) |
