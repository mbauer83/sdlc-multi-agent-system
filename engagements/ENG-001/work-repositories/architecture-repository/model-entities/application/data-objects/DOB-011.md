## PMDecision

`src/orchestration/pm_decision.py`. Pydantic model. The `result_type` for the PM Agent — the structured output that the PM Agent produces after each decision cycle. Used by the LangGraph routing function to select the next node.

Fields:
| Field | Type | Description |
|---|---|---|
| `next_action` | `Literal["invoke_specialist","evaluate_gate","surface_cqs","trigger_review","close_sprint","complete_engagement"]` | Primary routing key |
| `specialist_id` | `str \| None` | Agent role to invoke (when `next_action == "invoke_specialist"`) |
| `skill_id` | `str \| None` | Skill to execute on the specialist |
| `task_description` | `str` | Natural-language task description for the specialist |
| `reasoning` | `str` | PM's reasoning for this decision |
| `gate_id` | `str \| None` | Gate ID to evaluate (when `next_action == "evaluate_gate"`) |

The routing function reads `next_action` to select the next LangGraph node. `task_description` is included in the specialist's Layer 3 context.

## Properties

| Attribute | Value |
|---|---|
| Module | `src/orchestration/pm_decision.py` |
| Result type | PM Agent `result_type` |
| Consumer | LangGraph routing function (reads next_action) |
