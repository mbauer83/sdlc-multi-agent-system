## SDLCGraphState

**Primary use: route LangGraph execution and carry agent invocation context during one graph execution cycle.**

`src/orchestration/graph_state.py`. LangGraph `TypedDict` (not Pydantic — LangGraph requires TypedDict for graph state). The shared state object passed between all LangGraph nodes. Carries routing signals and immediate execution context only — it does NOT duplicate the full engagement history. For full engagement state (open CQs, gate outcomes, baselined artifacts, etc.) the nodes read from `WorkflowState` (DOB-009), which they receive via `AgentDeps.workflow_state`.

Fields:
| Field | Type | Description |
|---|---|---|
| `engagement_id` | `str` | Current engagement identifier |
| `current_agent` | `str \| None` | Agent role currently executing |
| `current_skill` | `str \| None` | Skill currently loaded |
| `pm_decision` | `PMDecision \| None` | Most recent PM decision (routing key) |
| `last_specialist_output` | `str \| None` | Output of the most recent specialist node |
| `target_repository_ids` | `list[str]` | Registered target repo IDs for this engagement |
| `primary_repository_id` | `str \| None` | Primary target repo ID |
| `review_pending` | `bool` | True when sprint review is outstanding |
| `algedonic_active` | `bool` | True when an active S1/S2 algedonic signal exists |

This TypedDict is the LangGraph state; `WorkflowState` (EventStore) is the canonical state. They are complementary, not redundant — SDLCGraphState carries routing context; WorkflowState carries full engagement history.
