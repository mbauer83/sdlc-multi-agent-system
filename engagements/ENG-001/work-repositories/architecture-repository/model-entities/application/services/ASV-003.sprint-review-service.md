## Sprint Review Service

Named service grouping: APP-018 (UserInteractionOrchestrator) + APP-021 (UserInputGateway) + APP-016 review_processing_node.

Full sprint review lifecycle:
1. PM node emits `review.pending` (sprint not yet closed)
2. DashboardServer (APP-020) surfaces `/review` page with artifact list
3. User marks each item (approved / needs-revision / rejected + agent_tag + comment)
4. `POST /review/submit` → UserInputGateway validates all items decided → `review.submitted`
5. UserInteractionOrchestrator signals `review_processing_node`
6. review_processing_node: rejected → `artifact.rejected`; needs-revision → `handoff.created` to tagged agent; approved → proceed → `review.sprint-closed` → `sprint.close`

## Properties

| Attribute | Value |
|---|---|
| Components | APP-018, APP-021, APP-016 (review_processing_node) |
| Event chain | `review.pending` → `review.submitted` → `review.sprint-closed` → `sprint.close` |
| Realises | CAP-005 (User Interaction — review half) |
