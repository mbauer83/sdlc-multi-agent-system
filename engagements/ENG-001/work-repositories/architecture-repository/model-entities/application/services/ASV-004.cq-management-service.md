## CQ Management Service

Named service grouping: APP-007 (PM — CQ batching) + APP-018 (UserInteractionOrchestrator — event monitoring) + APP-020 (DashboardServer — surfacing) + APP-021 (UserInputGateway — answer submission).

Full CQ lifecycle:
1. Specialist agent calls `raise_cq()` tool → `cq.raised` EventStore event
2. PM batches open CQs → `cq.batched`
3. DashboardServer serves `/queries` with open CQs (count badge updates via SSE)
4. User types answer + optional file upload → `POST /queries/<id>/answer`
5. UserInputGateway validates → `cq.answered`
6. UserInteractionOrchestrator routes → `cq.routed` to raising agent's node
7. Agent resumes with answer injected into context

## Properties

| Attribute | Value |
|---|---|
| Components | APP-007, APP-018, APP-020, APP-021 |

| Realises | CAP-005 (User Interaction — query half) |
