## Skill Execution

**Behavioral element.** The structured, sequenced application-layer interaction performed by the Skill Execution Collaboration (ACO-001). An ApplicationInteraction in ArchiMate is the behavioral counterpart to an ApplicationCollaboration — the collaboration (structure) is assigned to perform this interaction (behavior), and this interaction realizes the corresponding BusinessProcess (BPR-002).

### Execution sequence

The interaction is strictly sequenced. Steps must occur in order:

1. **Step 0.L — Learnings Lookup** (ASV-005): Query LearningStore for prior entries matching the current phase, artifact-type, and agent role. Inject retrieved learnings into the skill context.
2. **Step 0 — Discovery Scan** (ASV-002): Invoke `list_artifacts` and `search_artifacts` across the five discovery layers. Read the engagement profile and any referenced upstream artifacts.
3. **Steps 1–N — Skill Execution** (ASV-001): Assemble 4-layer system prompt (identity + behavioral stance + skill file + tool results). Execute the skill's prescribed steps in sequence. Call tools as instructed.
4. **Artifact write** (ASV-002): Produce output artifacts via `write_artifact()`. Each write emits an `artifact.created` or `artifact.updated` event via AIF-004.
5. **Feedback-loop close** (ASV-005): If a correction occurred or a significant architectural decision was made, generate a LearningEntry and persist it via AIF-006.

The sequence is non-negotiable: learnings lookup before discovery, discovery before skill steps, skill steps before artifact write, feedback-loop close after artifact write. A violation is an algedonic trigger (ALG-C01: skill execution out of sequence).
