## Sprint Review

At sprint close, the PM emits a `review.pending` event. The dashboard surfaces the sprint's artifact list to the user. The user marks each item as approved, needs-revision, or rejected (with optional agent tag and comment).

On submission, the `review_processing_node` routes needs-revision and rejected items as handoffs to the tagged specialist agent(s). Revised artifacts are written back to the repository and the user is presented with a fresh review of the updated artifacts. This revision cycle repeats until the user approves **all** sprint artifacts — no item may remain `needs-revision` or `rejected` when the sprint closes. The user has authority to request indefinite rework cycles; agents must not assume a single revision pass is sufficient.

Only once all items are approved does `sprint.close` proceed. The sprint is not closed while any revision cycle is outstanding.

## Properties

| Attribute | Value |
|---|---|
| Trigger | PM decides `trigger_review`; `review.pending` event emitted (sprint not closed yet) |
| Outcome | All artifacts user-approved → `review.submitted` → `review.sprint-closed` → `sprint.close` |
| Revision cycle | Repeat until all items approved: route corrections → revise → re-present to user |
| Safety-Relevant | No |
