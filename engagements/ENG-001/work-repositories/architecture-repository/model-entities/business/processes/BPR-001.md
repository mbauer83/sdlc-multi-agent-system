## Sprint Planning

The PM agent determines which specialist agents to invoke in the current sprint, assigns work packages, and emits the `sprint.started` event. Sprint planning considers current phase, open CQs, outstanding handoffs, and gate outcomes before dispatching the first specialist invocation.

## Properties

| Attribute | Value |
|---|---|
| Trigger | `engagement.started` event or `sprint.close` event (start of next sprint) |
| Outcome | Ordered list of specialist invocations for the sprint; `sprint.started` event recorded |
| Safety-Relevant | No |
