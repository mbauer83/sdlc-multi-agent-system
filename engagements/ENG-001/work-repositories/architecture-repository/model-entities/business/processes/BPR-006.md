## Sprint Review

At sprint close, the PM emits a `review.pending` event. The dashboard surfaces the sprint's artifact list to the user. The user marks each item as approved, needs-revision, or rejected (with optional agent tag and comment). On submission, the `review_processing_node` routes rejected items as rejection events and needs-revision items as handoffs to the tagged agent. Approved items proceed to `sprint.close`.

## Properties

| Attribute | Value |
|---|---|
| Trigger | PM decides `trigger_review`; `review.pending` event emitted (sprint not closed yet) |
| Outcome | `review.submitted` event; `review.sprint-closed` event; `sprint.close` event |
| Safety-Relevant | No |
