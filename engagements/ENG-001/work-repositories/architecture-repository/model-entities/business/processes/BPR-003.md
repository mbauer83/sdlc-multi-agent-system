## CQ Lifecycle

A specialist agent raises a clarification request when required information is absent from all available sources. The PM batches open CQs. The dashboard surfaces them to the user. The user provides an answer (and optionally uploads a file). The UserInteractionOrchestrator routes the answer back to the raising agent's node, which resumes with the answer in context.

## Properties

| Attribute | Value |
|---|---|
| Trigger | `cq.raised` event from a specialist agent tool call |
| Outcome | `cq.answered` event with user's answer text and optional upload reference; raising agent resumes |
| Safety-Relevant | No |
