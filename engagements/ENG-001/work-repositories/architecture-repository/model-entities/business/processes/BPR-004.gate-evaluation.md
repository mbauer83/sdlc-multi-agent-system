## Gate Evaluation

The PM collects gate votes from all relevant agents (including a mandatory CSCO vote) and evaluates the phase-transition checklist. A CSCO veto blocks transition regardless of other votes. The outcome (passed / failed / deferred) is recorded as a `gate.evaluated` event. On pass, a `phase.transitioned` event is emitted.

## Properties

| Attribute | Value |
|---|---|
| Trigger | PM decides `evaluate_gate` after all required specialist outputs are complete |
| Outcome | `gate.evaluated` event with all votes and result; `phase.transitioned` on pass |
| Safety-Relevant | No |
