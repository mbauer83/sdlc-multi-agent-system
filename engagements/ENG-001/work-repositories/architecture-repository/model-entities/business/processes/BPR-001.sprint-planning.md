## Sprint Planning

PM determines specialist invocations for the current sprint and emits sprint.started for active engagements.

## Properties

| Attribute | Value |
|---|---|
| Guards | Must not run when engagement is paused (BEV-009 active) or completed (BEV-008 recorded) |
| Outcome | Ordered specialist invocation plan and sprint.started event |
| Safety-Relevant | No |
| Trigger | engagement.started, sprint.close, or engagement.resumed (BEV-010) |

## Notes

Resume path is explicit: BEV-010 clears pause state and re-enables planning.
