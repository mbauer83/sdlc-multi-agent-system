## Process Review Decisions

The PM routes needs-revision and rejected items as handoffs to the tagged specialist agents, who revise and resubmit. The revision cycle repeats (returning to BPR-601) until all items are approved. When all artifacts are approved, the PM emits `sprint.close`, and Sprint Planning (BPR-001) is triggered for the next sprint.

## Properties

| Attribute | Value |
|---|---|
| Input | Per-artifact review decisions from BPR-602 |
| Output | Handoffs to specialists for revisions (if any); or `sprint.close` event + trigger to BPR-001 when all approved |
| Loops-to | BPR-601 (if any items need revision) |
| Triggers | BPR-001 (Sprint Planning) when sprint is fully approved and closed |
