## Record Gate Outcome

The PM writes the GateOutcome (BOB-004) to the work-repository and emits the `gate.evaluated` event. On pass, a `phase.transitioned` event is also emitted and BPR-006 (Sprint Review) is triggered. On fail or defer, the blocking items are queued for the next sprint planning cycle and BPR-001 is re-triggered without phase advance.

## Properties

| Attribute | Value |
|---|---|
| Input | Gate evaluation result (from BPR-402) |
| Output | GateOutcome (BOB-004) written; `gate.evaluated` event; `phase.transitioned` event (on pass) |
| Triggers (on pass) | BPR-006 (Sprint Review) |
| Triggers (on fail/defer) | BPR-001 (Sprint Planning) — re-plan without phase advance |
