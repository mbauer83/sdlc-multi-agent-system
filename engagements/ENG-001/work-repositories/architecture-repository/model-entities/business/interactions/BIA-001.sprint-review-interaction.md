## Sprint Review Interaction

The structured multi-party governance interaction where PM and User review sprint outputs, decide continue/end/pause, and drive rework cycles until closure criteria are met.

## Properties

| Attribute | Value |
|---|---|
| Assigned To | BCO-002 (Sprint Review Board) |
| Decision Outcomes | Continue -> BEV-002, End -> BEV-008, Pause -> BEV-009 |
| Outer-Level Semantics | Canonical Sprint Review representation for operational governance |
| Realizes | BSV-009 (User Decisions Service) |
| Resume Path | Paused engagement resumes through BEV-010, which re-triggers BPR-001 |
| Sequence Steps | BIA-101 PM presents -> BIA-102 User reviews/decides -> BIA-103 PM processes decision and routes outcomes |
| Trigger | BPR-004 (Gate Evaluation passed) and BEV-006 (Sprint Review Submitted) |
| User Role | Mandatory: PM cannot finalize continue/end/pause without explicit user decision |

## Notes

Sub-interactions BIA-101/102/103 are the only decomposition of Sprint Review in this model.
