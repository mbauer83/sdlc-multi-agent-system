## Sprint Review Interaction

The structured, multi-party behavioral sequence through which the PM Agent and User jointly evaluate and close a sprint. The interaction proceeds in a defined sequence: (1) PM presents the sprint output summary via Dashboard; (2) User reviews each item and selects a decision (approve / approve-with-corrections / re-run); (3) User submits the review (`BEV-006` raised); (4) PM processes the review decision, applies corrections or re-runs flagged items, and either closes the sprint or iterates. Neither participant can complete this sequence alone — this is a genuine collaborative interaction, not parallel individual work.
