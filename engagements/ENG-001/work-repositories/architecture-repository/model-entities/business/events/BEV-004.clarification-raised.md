## Clarification Raised

Raised by a specialist agent during skill execution when a knowledge gap is detected that requires human input. This event suspends the affected skill step, routes the question to the user via the Human Collaboration function (BFN-003), and initiates the CQ Lifecycle process (BPR-003). Blocking CQs halt the invoking skill until answered; non-blocking CQs allow the skill to proceed with a documented assumption.

## Properties

| Attribute | Value |
|---|---|
| Form | `cq.raised` EventStore event |
| Triggers | BPR-003 (CQ Lifecycle) |
| Emitter | Any specialist agent via `raise_cq` tool call |
| VS Mapping | Can occur in any VS-001 stage; concentrated in Stages 2–3; critical in VS-002 Stage 4 (Model Confirmation) |
