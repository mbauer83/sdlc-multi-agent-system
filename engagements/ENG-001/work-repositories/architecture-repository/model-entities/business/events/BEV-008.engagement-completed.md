## Engagement Completed

The terminal event that ends the engagement lifecycle. Raised by the EngagementSession when the last phase or VS stage successfully concludes — specifically after Enterprise Promotion (BPR-007) completes for VS-001 engagements, or after Governance Activation for VS-002 engagements. This event terminates the Sprint Planning loop: BPR-001 must NOT re-fire after BEV-008 is recorded in the EventStore.

## Properties

| Attribute | Value |
|---|---|
| Form | `engagement.completed` EventStore event |
| Terminates | BPR-001 loop (Sprint Planning no longer re-fires) |
| Emitter | EngagementSession after final phase/promotion concludes |
| VS Mapping | VS-001: fired after Stage 6 (Architecture Promotion); VS-002: fired after Stage 5 (Governance Activation) |

## Stop Conditions

This event, together with the PM sprint-planning decision logic, defines the engagement termination boundaries:

| Condition | PM Decision |
|---|---|
| No further phases remain in engagement scope | Record `engagement.completed`; do not plan a next sprint |
| BPR-007 (Enterprise Promotion) completes successfully | Check if any remaining phase obligations exist; if none, raise BEV-008 |
| User explicitly terminates the engagement | PM records `engagement.completed` with `termination-reason: user-requested` |
| All Must-priority requirements have `●` RTM entries and all phase gates have passed | Natural completion trigger |

Note: BEV-002 (Sprint Closed) does NOT trigger BPR-001 if BEV-008 has already been raised. The orchestration layer uses EventStore state to guard against re-entry after completion.
