## Sprint Closed

Raised when all specialist invocations in the current sprint have completed and the sprint output has been committed to the work-repositories. This event is the heartbeat of the engagement — it triggers the next planning cycle and determines whether the phase continues, advances, or pauses for review.

## Properties

| Attribute | Value |
|---|---|
| Form | `sprint.close` EventStore event |
| Triggers | BPR-001 (Sprint Planning — next sprint); BPR-006 (Sprint Review) when sprint-review config is enabled |
| Emitter | sprint_close_node (LangGraph orchestration layer) |
| VS Mapping | Transition events within any VS-001 or VS-002 stage |
