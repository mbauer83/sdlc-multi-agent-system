## Sprint Review Submitted

Raised when the user submits their sprint review decision via the Dashboard — either approving the sprint output as-is, approving with corrections, or requesting a re-run of specific items. This event transitions the Sprint Review interaction (BIA-001) from "review pending" to "review processing" and triggers PM to reconcile the review decision against the sprint artifacts.

## Properties

| Attribute | Value |
|---|---|
| Form | `review.submitted` EventStore event |
| Triggers | BPR-006 (Sprint Review — processing phase) |
| Emitter | Dashboard user-interaction handler → UserInteractionOrchestrator |
| VS Mapping | Occurs at end of any VS-001 stage sprint where sprint-review is enabled |
