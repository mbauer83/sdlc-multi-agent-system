## Engagement Resumed

Raised when a paused engagement is explicitly resumed and governance flow is re-enabled.

## Properties

| Attribute | Value |
|---|---|
| Decision Source | User/PM resume decision via governance interaction and PM routing |
| Effect | Clears paused state and re-enables BPR-001 (Sprint Planning) |
| Form | phase.resumed EventStore event |
| Precondition | BEV-009 (Engagement Paused) is active |

## Notes

Resume is mandatory counterpart of pause; engagement state remains intact across suspend/resume.
