## Learning Entry

A structured correction or validated insight generated by a specialist agent following a skill invocation that produced an error, gate veto, or reviewer-required revision. Each Learning Entry records: what went wrong (or what was confirmed as correct), why it happened, and how to avoid or repeat it in future invocations. Learning Entries are the primary mechanism by which individual agents improve over time without human intervention.

## Properties

| Attribute | Value |
|---|---|
| Lifecycle | Generated at end of BPR-002 (Skill Execution) when feedback-revision, gate-veto, or algedonic trigger fires; stored in learning repository; queried at start of each subsequent BPR-002 invocation (Step 0.L) |
| VS Stage Coverage | Generated in VS-001 Stages 2–5 (wherever skill execution occurs); promoted to enterprise learning corpus in Stage 6 |
| Application Layer Counterpart | DOB-003 (LearningEntry) — carries agent, phase, artifact-type, error-type, importance, trigger-text, correction-text, related entries |
