## Architecture Artifact

A versioned, identifiable document or model file produced by a specialist agent during skill execution and stored in the engagement work-repositories. Architecture Artifacts are the durable record of all phase work: they survive sprint boundaries, are promoted through gate evaluations, and serve as the primary inputs to downstream phase skills. Their lifecycle spans creation → draft → baselined → promoted.

## Properties

| Attribute | Value |
|---|---|
| Lifecycle | Created during BPR-002 (Skill Execution); evaluated during BPR-004 (Gate Evaluation); promoted during BPR-007 (Enterprise Promotion) |
| VS Stage Coverage | Produced across VS-001 Stages 2–6; reconstructed in VS-002 Stages 1–3 |
| Application Layer Counterpart | DOB-013 (ArtifactRecord) — records provenance, version, path, and status |

## Concept Map Entry

Every Architecture Artifact at the business layer corresponds to a `DOB-013` record at the application layer, which carries the metadata needed to manage and retrieve the artifact.
