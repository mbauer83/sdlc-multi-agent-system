# ADR-001: PydanticAI for Agent Definition

## Context
The runtime needs strongly-typed specialist agents, deterministic tool invocation, and structured outputs for PM and specialist roles. Candidate approaches considered: LangChain, Autogen, and direct raw API orchestration.

## Decision
Use PydanticAI as the primary agent-definition framework.

## Rationale
- Structured outputs through result typing support contract-driven orchestration.
- Native tool-calling model aligns with the existing MCP and repository tool surfaces.
- Minimal hidden framework behavior improves observability and governance alignment.
- Direct compatibility with Anthropic model usage in this repository.

## Consequences
- Tight coupling to Pydantic v2 and PydanticAI APIs.
- Runtime wrappers remain thinner and more maintainable than heavier orchestration abstractions.
- Migration effort required if upstream framework semantics materially change.
