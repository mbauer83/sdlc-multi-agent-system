# ADR-004: FastAPI + Jinja2 + SSE for Dashboard

## Context
The user interaction surface must support read views, CQ answering, uploads, and sprint review flow without introducing a separate frontend build system.

## Decision
Use FastAPI with server-side Jinja2 templates and SSE for live update signaling.

## Rationale
- Python-native stack matches repository implementation language and skills.
- Server-side rendering keeps deployment simple and transparent.
- SSE is sufficient for low-frequency state updates without websocket complexity.
- Form-post interaction model satisfies CQ/review workflows with minimal client code.

## Consequences
- UI interactivity remains intentionally limited versus SPA frameworks.
- Small amount of browser JavaScript is still required for review-toolbar behavior.
- Real-time throughput assumptions should be re-evaluated if interaction volume grows significantly.
