# ADR-005: File-Based ERP Entity Storage

## Context
The architecture model requires human-reviewable, versioned artifacts with deterministic validation and query behavior across engagements.

## Decision
Store one markdown file per entity/connection/diagram using ERP v2.0 conventions and build model indexes from filesystem frontmatter.

## Rationale
- Git-native diff/blame history for architectural changes.
- No schema migration overhead for model evolution as long as conventions are respected.
- Human-readable artifacts simplify governance review and cross-role handoff.
- Query and verifier engines can rebuild state deterministically from source files.

## Consequences
- Startup indexing time grows with corpus size.
- Requires robust verifier and query tooling to prevent model drift.
- Filesystem watcher and incremental refresh strategy are needed for responsive runtime behavior at scale.
