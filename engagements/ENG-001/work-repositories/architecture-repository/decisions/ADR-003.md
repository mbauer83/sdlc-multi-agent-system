# ADR-003: SQLite EventStore as Canonical Workflow State

## Context
The framework needs durable, auditable workflow history with deterministic replay and local-only operation for engagement portability.

## Decision
Use SQLite-backed EventStore as the canonical state source, with YAML event projection as a human-readable companion.

## Rationale
- ACID guarantees and single-file locality fit engagement-scoped execution.
- Replay model supports deterministic state reconstruction from event history.
- Local runtime operation avoids mandatory external infrastructure dependencies.
- Git-tracked workflow database preserves audit continuity inside the framework repo.

## Consequences
- Single-writer assumptions must be preserved in orchestration logic.
- Binary file diffs are less reviewable than plain-text artifacts.
- Snapshotting and integrity checks are required to bound replay costs over time.
